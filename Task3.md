When a patient submits the consultation form on the OrthoNow landing page, the browser first validates the required fields (Name, Phone, and Clinic Preference). On successful submission, the page immediately pushes the consultation_form_submitted event to the dataLayer, which Google Tag Manager captures and forwards to GA4. The event is marked as a conversion and imported into Google Ads so Smart Bidding can optimise campaigns based on completed consultation enquiries.
At the same time, the landing page sends the form data through a secure HTTPS POST request to a backend API. I would use a direct integration with the HubSpot CRM API rather than Zapier or Make because it provides better performance, reliability, and control while reducing dependency on third-party automation platforms. The backend checks whether the submitted phone number already exists in HubSpot. If it does, the existing contact is updated; otherwise, a new contact is created with the fields: Name, Phone, Clinic Preference, Source = "Google Ads - Consultation Landing Page", and Lead Status = "New Enquiry".
Once HubSpot confirms a successful response, the backend calls the Karix WhatsApp Business API to send a confirmation message to the client. Separating CRM creation and WhatsApp delivery into individual API calls allows each process to be monitored and retried independently if necessary.
The biggest failure point is the HubSpot API being unavailable or returning an error. If contact creation fails, the lead should not be lost. Instead, the backend should temporarily store the enquiry in a retry queue or database with a status of "Pending Sync". A background worker can retry the HubSpot request using exponential backoff until it succeeds, while logging all failures for operational visibility.
The requirement to send the WhatsApp message within two minutes could be affected by API outages, network latency, server overload, or delays in processing queued requests. To monitor this SLA, I would log timestamps for form submission and WhatsApp delivery, create alerts if delivery exceeds two minutes, and use application monitoring dashboards (such as Grafana or Datadog) to track API response times, error rates, queue length, and message delivery success. This ensures operational issues are detected quickly and resolved before they affect patients.
